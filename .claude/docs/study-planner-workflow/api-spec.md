# API Specification

## Overview
This document outlines the specification for the API used in the Structured Development workflow. The API facilitates interactions with resources, providing endpoints for CRUD operations and validation processes.

**Base URL**: `/api/v1`  
**Authentication**: Bearer Token / API Key

## Endpoints

### Resource Management

#### GET /resource
Retrieve a list of resources.

**Query Parameters:**
| Parameter | Type   | Required | Description                           |
|-----------|--------|----------|---------------------------------------|
| page      | number | No       | Page number for paginated results (default: 1) |
| limit     | number | No       | Number of items per page (default: 20) |

**Response:**
```json
{
  "data": [
    {
      "id": "1",
      "name": "Resource One",
      "createdAt": "2023-01-01T00:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 100
  }
}
```

#### GET /resource/:id
Retrieve a single resource by its ID.

**Path Parameters:**
| Parameter | Type   | Description         |
|-----------|--------|---------------------|
| id        | string | Unique resource ID  |

**Response:**
```json
{
  "id": "1",
  "name": "Resource One",
  "description": "This is a sample resource.",
  "createdAt": "2023-01-01T00:00:00Z",
  "updatedAt": "2023-01-02T00:00:00Z"
}
```

**Error Responses:**
| Status | Description            |
|--------|------------------------|
| 404    | Resource not found     |

#### POST /resource
Create a new resource.

**Request Body:**
```json
{
  "name": "string (required)",
  "description": "string (optional)"
}
```

**Response:**
```json
{
  "id": "1",
  "name": "Resource One",
  "createdAt": "2023-01-01T00:00:00Z"
}
```

**Error Responses:**
| Status | Description                    |
|--------|--------------------------------|
| 400    | Invalid request body           |
| 409    | Resource already exists        |

#### PUT /resource/:id
Update an existing resource.

**Path Parameters:**
| Parameter | Type   | Description         |
|-----------|--------|---------------------|
| id        | string | Unique resource ID  |

**Request Body:**
```json
{
  "name": "string (optional)",
  "description": "string (optional)"
}
```

**Response:**
```json
{
  "success": true,
  "message": "Resource updated successfully."
}
```

#### DELETE /resource/:id
Delete a resource.

**Path Parameters:**
| Parameter | Type   | Description         |
|-----------|--------|---------------------|
| id        | string | Unique resource ID  |

**Response:**
```json
{
  "success": true,
  "message": "Resource deleted successfully."
}
```

## Data Models

### Resource
```typescript
interface Resource {
  id: string;              // Unique identifier for the resource
  name: string;            // Name of the resource
  description?: string;    // Optional description of the resource
  createdAt: string;       // Timestamp of resource creation
  updatedAt: string;       // Timestamp of last update
}
```

## Error Format
All error responses conform to the following structure:
```json
{
  "error": {
    "code": "ERROR_CODE",          // Code representing the error
    "message": "Human readable message", // Description of the error
    "details": {}                   // Optional field for additional error details
  }
}
```

## Rate Limiting
- **Limit**: 100 requests per minute
- **Headers**:
  - `X-RateLimit-Limit`: Maximum number of requests allowed
  - `X-RateLimit-Remaining`: Number of requests remaining in the current window
